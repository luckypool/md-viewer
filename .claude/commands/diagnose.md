# /diagnose - 問題診断コマンド

問題の原因を調査するためのワークフローコマンドです。

## 使用方法

```
/diagnose <問題の症状>
```

例:
- `/diagnose Google Picker が開かない`
- `/diagnose ビルドエラーが発生する`
- `/diagnose Markdown が正しく表示されない`

## ワークフロー

### Step 1: 症状の確認

- エラーメッセージの有無
- 発生条件 (いつ、どのタイミングで)
- 再現性 (常に発生 / 時々発生)

### Step 2: 原因の仮説

- コードの問題
- 設定の問題
- 環境の問題
- 外部サービスの問題

### Step 3: 調査

```bash
# ビルドエラーの場合
npm run build

# Lint エラーの場合
npm run lint

# 実行時エラーの場合
npm run dev  # ブラウザのコンソールを確認
```

### Step 4: 原因の特定

関連するコードを読み、問題の箇所を特定

### Step 5: 解決策の提案

- 修正方法
- 確認手順

## よくある問題

### Google API 関連

| 症状 | 可能性のある原因 |
|------|-----------------|
| Picker が開かない | API キーの設定ミス、スクリプト未ロード |
| 認証エラー | Client ID の設定ミス、OAuth 設定 |
| ファイル取得失敗 | スコープ不足、API 未有効化 |

### ビルド関連

| 症状 | 可能性のある原因 |
|------|-----------------|
| 型エラー | 型定義の不整合、import ミス |
| モジュールエラー | パッケージ未インストール、パス設定 |
| ビルド失敗 | Vite 設定、TypeScript 設定 |

### 表示関連

| 症状 | 可能性のある原因 |
|------|-----------------|
| スタイル崩れ | CSS の競合、クラス名ミス |
| Markdown 表示異常 | プラグイン設定、コンテンツ形式 |
| ハイライト失敗 | 言語指定ミス、プラグイン問題 |

## 出力フォーマット

```markdown
## 診断結果

### 症状
[問題の説明]

### 原因
[特定された原因]

### 解決策
1. [手順1]
2. [手順2]

### 確認方法
[問題が解決したことの確認手順]
```
