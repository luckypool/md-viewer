# /fix - バグ修正コマンド

バグを修正するためのワークフローコマンドです。

## 使用方法

```
/fix <問題の説明>
```

## ワークフロー

1. **問題の特定**
   - エラーメッセージを確認
   - 再現手順を理解
   - 関連するコードを特定

2. **原因調査**
   - コードを読んで原因を特定
   - 型エラー、ロジックエラー、状態管理の問題など

3. **修正**
   - 最小限の変更で修正
   - 副作用がないことを確認

4. **検証**
   - ビルド確認 (`npm run build`)
   - Lint 確認 (`npm run lint`)
   - 問題が解決したことを確認

## よくある問題と対処

### TypeScript エラー

```typescript
// 型の不一致
// 原因: Props の型定義とコンポーネントの実装が不一致
// 対処: 型定義を修正、または型ガードを追加

// null/undefined エラー
// 原因: optional な値を non-null として扱っている
// 対処: Optional chaining (?.) や nullish coalescing (??) を使用
```

### React エラー

```typescript
// Invalid hook call
// 原因: フックがコンポーネント外で呼ばれている
// 対処: フックの呼び出し位置を確認

// Too many re-renders
// 原因: 無限ループの状態更新
// 対処: useEffect の依存配列を確認
```

### Google API エラー

```typescript
// gapi is not defined
// 原因: スクリプトがロードされる前にアクセス
// 対処: ロード完了を待つ処理を追加

// OAuth エラー
// 原因: 認証情報の問題
// 対処: .env の設定を確認
```
