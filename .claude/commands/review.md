# /review - コードレビューコマンド

コードの品質をチェックするためのワークフローコマンドです。

## 使用方法

```
/review [ファイルパス]
```

ファイルパスを指定しない場合は、最近変更されたファイルをレビューします。

## レビュー観点

### 1. 型安全性

- [ ] `any` の使用がないか
- [ ] 適切な型定義がされているか
- [ ] null/undefined が適切に処理されているか

### 2. React ベストプラクティス

- [ ] 関数コンポーネントを使用しているか
- [ ] Hooks のルールに従っているか
- [ ] 不要な再レンダリングがないか
- [ ] メモ化が適切に使われているか

### 3. コード品質

- [ ] DRY 原則に従っているか
- [ ] 関数/コンポーネントが単一責任か
- [ ] 命名が明確か
- [ ] 不要なコードがないか

### 4. エラーハンドリング

- [ ] API エラーが適切に処理されているか
- [ ] ユーザーにエラーが通知されるか
- [ ] エッジケースが考慮されているか

### 5. セキュリティ

- [ ] 機密情報がハードコードされていないか
- [ ] XSS 対策がされているか

## レビュー出力フォーマット

```markdown
## レビュー結果: [ファイル名]

### ✅ 良い点
- ...

### ⚠️ 改善提案
- **行XX**: [問題の説明]
  - 現状: `コード`
  - 提案: `改善案`

### 🔴 要修正
- **行XX**: [問題の説明]
  - 理由: ...
  - 修正方法: ...
```

## 自動チェック

レビュー前に以下を実行:

```bash
npm run lint
npm run build
```
