# /implement - 機能実装コマンド

新しい機能を実装するためのワークフローコマンドです。

## 使用方法

```
/implement <機能説明>
```

## ワークフロー

1. **要件分析**
   - 機能の規模を判断 (小/中/大)
   - 影響範囲を特定
   - 必要なファイルをリストアップ

2. **設計 (中規模以上)**
   - コンポーネント設計
   - データフロー設計
   - 型定義

3. **実装**
   - 型定義から開始
   - コンポーネント/フック実装
   - スタイル調整

4. **品質チェック**
   - TypeScript 型チェック (`npm run build`)
   - ESLint (`npm run lint`)
   - 動作確認

5. **コミット**
   - 変更内容の要約
   - コミットメッセージ作成

## 規模判断の基準

| 規模 | 基準 | 対応 |
|------|------|------|
| 小 | 1-2ファイル、単純な変更 | 直接実装 |
| 中 | 3-5ファイル、新規コンポーネント | 簡易設計 → 実装 |
| 大 | 5+ファイル、アーキテクチャ変更 | 詳細設計 → 実装 |

## チェックリスト

- [ ] 既存コードのスタイルに従う
- [ ] 適切な型定義
- [ ] エラーハンドリング
- [ ] ビルドが通る
- [ ] Lint エラーなし
