# なぜ MarkDrive を作ったのか

このドキュメントでは、MarkDrive が解決する課題と提供する価値について説明します。

## 解決する課題

### Google Drive には Markdown プレビューがない

Google Drive は優れたクラウドストレージですが、Markdown ファイルのネイティブプレビュー機能がありません。`.md` ファイルを開くと、テキストエディタで生のマークアップが表示されるだけです。

技術ドキュメント、README、設計書などを Markdown で書いている人にとって、これは不便です。

### 既存ツールのプライバシー問題

多くのオンライン Markdown ビューアは、ファイルをサーバーにアップロードして処理します。これには以下の問題があります:

- **機密情報の漏洩リスク**: 社内ドキュメントや個人的なメモがサードパーティのサーバーに送信される
- **データの永続化**: アップロードされたファイルがどこかに保存される可能性
- **企業ポリシーへの抵触**: 多くの企業は外部サービスへのファイルアップロードを禁止している

### 見た目と機能の両立

Markdown を書く人は、美しいレンダリングを求めています:

- シンタックスハイライトされたコードブロック
- 正しく表示されるテーブル
- Mermaid によるダイアグラム

しかし、これらすべてをサポートしつつ、プライバシーを守るツールは多くありません。

## MarkDrive が提供する価値

### 1. 完全なプライバシー保護

MarkDrive は**サーバーレス設計**です。

```
Google Drive → ブラウザ（直接）
```

ファイルは Google Drive からブラウザに直接送信されます。中間サーバーは介在しません。つまり:

- ファイル内容が第三者に見られることはない
- データが外部に保存されることはない
- 企業のセキュリティポリシーに抵触しない

### 2. シームレスな Google Drive 連携

Google Drive に保存した Markdown ファイルを:

- **検索**: ファイル名で素早く検索
- **プレビュー**: ワンクリックで美しくレンダリング
- **共有**: PDF として出力して配布

追加のセットアップは最小限。Google アカウントでログインするだけで使い始められます。

### 3. プロフェッショナルなレンダリング

技術ドキュメントに必要な機能をすべてサポート:

| 機能 | 用途 |
|------|------|
| GFM | テーブル、タスクリスト、打ち消し線 |
| シンタックスハイライト | コードの可読性向上 |
| Mermaid | フローチャート、シーケンス図、ER図 |

### 4. どこでも同じ体験

- ブラウザさえあれば動作
- インストール不要
- 設定は自動的に同期（localStorage）

## 誰のためのツールか

### エンジニア・開発者

- README やドキュメントを Google Drive で管理している
- コードブロックやダイアグラムを含むドキュメントを書いている
- チームメンバーと技術ドキュメントを共有したい

### テクニカルライター

- Markdown で技術文書を執筆している
- レビュー用に PDF を生成したい
- 複数のドキュメントを素早く切り替えて確認したい

### プライバシーを重視する人

- 機密情報を含むドキュメントを扱っている
- 外部サービスへのファイルアップロードを避けたい
- データの取り扱いについて透明性を求めている

### チームリーダー・マネージャー

- チームのナレッジを Google Drive で一元管理している
- セキュリティポリシーを遵守しながらツールを使いたい
- メンバーに安全なツールを推奨したい

## ユースケース

### ケース 1: 技術ドキュメントのレビュー

> エンジニアの田中さんは、設計書を Markdown で書いて Google Drive に保存しています。レビューミーティングの前に、MarkDrive で美しくレンダリングされた状態を確認し、PDF として出力してチームに共有します。

### ケース 2: 個人のナレッジベース

> フリーランスの山田さんは、技術メモやTipsを Markdown で Google Drive に蓄積しています。必要なときに MarkDrive で素早く検索・閲覧し、情報にアクセスします。

### ケース 3: 社内ドキュメントの閲覧

> 大手企業の佐藤さんは、社内ポリシーにより外部サービスへのファイルアップロードが禁止されています。MarkDrive はサーバーにファイルを送信しないため、安心して利用できます。

## 設計思想

### プライバシーファースト

機能を追加するたびに「これはユーザーのプライバシーを損なわないか？」を最優先で確認します。便利さとプライバシーのトレードオフでは、常にプライバシーを選択します。

### シンプルさ

必要最小限の機能に絞り、複雑さを排除します。Markdown を美しく表示する。それだけに集中したツールです。

### 透明性

ソースコードは公開されています。データの取り扱いについて[プライバシードキュメント](./privacy.md)で詳細に説明しています。

---

## まとめ

MarkDrive は「Google Drive の Markdown を、プライバシーを守りながら美しく表示する」という一つのことに集中したツールです。

既存のソリューションが抱える問題（プレビュー機能の欠如、プライバシーの懸念、機能不足）を解決し、エンジニアやテクニカルライターが安心して使えるツールを目指しています。
